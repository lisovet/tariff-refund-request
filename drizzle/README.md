# Drizzle migrations

This directory holds generated SQL migrations. Generated by `npm run db:generate`, applied by `npm run db:migrate`.

## Workflow

1. Add or modify a table in `src/shared/infra/db/schema.ts` (and the per-context schema files registered there).
2. Run `npm run db:generate` — drizzle-kit emits a numbered SQL file here.
3. Review the SQL diff carefully (especially destructive changes).
4. Commit the migration alongside the schema change.
5. Apply with `npm run db:migrate` (CI does this automatically; locally against your dev branch).

## Per-environment strategy (per ADR 003)

| Environment | Database |
| --- | --- |
| Local dev | Neon dev branch (one per developer) — `DATABASE_URL` in `.env.local` |
| PR preview | Neon ephemeral branch per PR |
| Staging | Neon staging branch |
| Production | Neon prod tier or AWS RDS (per ADR 003 — TBD by ops) |

`DATABASE_URL` is read by `src/shared/infra/db/client.ts` and `drizzle.config.ts`.

## Test isolation

Integration tests use a real Postgres (Neon CI branch in CI, testcontainers locally). Each Vitest worker writes to its own schema (`pg_temp_<workerId>`) so parallel runs don't collide. Wiring lands in task #6.
